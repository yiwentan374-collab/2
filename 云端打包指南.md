# 云端自动生成 exe ·  纯网页操作指南（本地零安装）

不用装 Python、不用敲命令。全程在浏览器里点几下，GitHub 会在云端的 Windows 机器上帮你生成 exe，你下载下来双击就能用。

预计用时：10～15 分钟，主要是注册账号和等编译。

---

## 准备：把项目解压好

先把 `desktop_pet.zip` 解压出来，记住 `desktop_pet` 这个文件夹的位置。里面应该能看到 `app.py`、`assets` 文件夹、`pet.spec`、以及一个 `.github` 文件夹（如果 Windows 没显示 `.github`，在资源管理器上方"查看"里勾选"隐藏的项目"）。

---

## 第 1 步：注册 / 登录 GitHub

1. 打开 https://github.com
2. 右上角 **Sign up** 注册（用邮箱即可，免费），或已有账号就 **Sign in**

---

## 第 2 步：新建一个仓库（repository）

1. 登录后点右上角的 **+** → **New repository**
2. **Repository name** 随便填，比如 `bichon-pet`
3. 选 **Public**（公开，免费额度足够；私有也行）
4. 其它都不用动，点最下面绿色按钮 **Create repository**

---

## 第 3 步：上传项目文件

1. 新仓库页面中间有一行小字链接：**uploading an existing file**，点它
   （或者页面里 **Add file → Upload files**）
2. 打开你解压出的 `desktop_pet` 文件夹，**全选里面所有内容**（`app.py`、`assets`、`pet.spec`、`build.bat`、`.github` 等），整体**拖进**网页那个上传框
   - 注意：拖的是文件夹**里面的东西**，不要把最外层 `desktop_pet` 文件夹本身拖进去
   - `.github` 文件夹一定要一起传上去（它是自动编译的关键）
3. 等文件都列出来后，点最下面绿色按钮 **Commit changes**

---

## 第 4 步：等云端自动编译

1. 上传完成后，点页面顶部的 **Actions** 标签
2. 会看到一条正在运行的任务（黄色圆点转圈），名字是 **Build Windows EXE**
3. 等它变成**绿色对勾**（大约 1～3 分钟）。如果是红色叉，看文末"出错怎么办"

---

## 第 5 步：下载 exe

1. 点那条已完成（绿勾）的任务进去
2. 拉到页面**最底部**，有个 **Artifacts** 区域，里面有 **BichonPet-exe**
3. 点它下载（下载下来是个 zip）
4. 解压，里面就是 **`BichonPet.exe`** —— 双击运行，你的桌宠就出来了！

> 这个 exe 可以拷到任何 Windows 电脑用，对方也不需要装 Python。

---

## 以后想改东西

改完文件（比如换 GIF、改台词），重新走第 3 步上传覆盖，GitHub 会自动重新编译，再按第 5 步下载新 exe 即可。

---

## 出错怎么办

- **Actions 任务变红色叉**：点进去看哪一步红了，把红色那段文字截图发我。最常见是 `.github` 文件夹没传上去（那样根本不会触发编译，Actions 里空空的）——确认仓库里能看到 `.github/workflows/build.yml`。
- **看不到 Actions 在跑**：多半是 `.github` 没上传成功。回第 3 步，确认这个文件夹和里面的 `build.yml` 都在仓库里。
- **下载的 exe 双击闪退**：看 exe 同目录是否生成了 `pet_error.log`，把内容发我。
- **杀毒软件报警**：PyInstaller 打的单文件 exe 常被误报，加入信任即可。
